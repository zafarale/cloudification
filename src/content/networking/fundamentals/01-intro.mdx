Computer networking is the foundation of how modern systems communicate. Whether you're building web applications, deploying cloud infrastructure, or debugging distributed systems, understanding networking fundamentals is essential.

## What is a Network?

A **network** is a collection of devices (computers, servers, routers, switches) connected together to share resources and communicate. Networks can be as small as two devices connected by a cable, or as large as the entire internet.

## Why Learn Networking?

- **Debugging** — Most application issues involve the network at some level
- **Security** — Understanding how data flows helps you protect it
- **Performance** — Network latency often dominates application response times
- **Cloud** — Modern infrastructure is fundamentally networked

## The OSI Model Preview

We'll cover the OSI model in depth, but here's a quick overview of how data moves through network layers:

```
Application  (HTTP, DNS, etc.)
     ↓
Transport    (TCP, UDP)
     ↓
Network      (IP, routing)
     ↓
Data Link    (Ethernet, MAC)
     ↓
Physical     (cables, signals)
```

Each layer has a specific responsibility, and understanding this separation helps you reason about where problems occur.
